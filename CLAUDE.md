# kickxkick 開発ルール

## Source of truth

現在の正式な仕様は `specs/KICKXKICK_*` にあります。
古いSoleMuseum時代の文書は、現在の仕様として扱わないでください。

## トークン節約ルール

毎回プロジェクト全体を要約しないでください。以下を使い分けます。

- コード = 現在の実装状態
- `specs/KICKXKICK_*` = 決まっている製品仕様
- `docs/AI_RESUME.md` = 直近の引き継ぎポイントのみ

## 操作の自動承認・要確認ルール

### 自動で進める（確認不要）
- コード修正・ファイル編集
- 調査・コード読み込み・検索
- git commit

### 確認必須 ＋ Discord通知

以下に該当する場合は、**まず Discord に「承認待ち」通知を送り、その後チャットで確認を求める**。
承認されてから実行する。（Webhook URL はグローバル設定に保存済み）

- **DBマイグレーション**（スキーマ変更・ALTER TABLE等）
- **git push**
- **大きな設計変更**（アーキテクチャ・データ構造に関わる判断）
- **削除系の操作**（ファイル削除・データ削除・DROP TABLE等）
- **仕様の変更**（specs/KICKXKICK_*の内容を変える判断）
- **仕様の解釈に迷う場面**

#### 手順（必ずこの順で）
1. Discord に通知（「承認待ち」の内容）
2. チャットで確認を求める
3. ユーザーが承認したら実行

## セッション終了時

`docs/AI_RESUME.md` を「前回やったこと／Next／Blocked」の形式で、最小限の内容に更新する。
求められない限り、長い報告は書かない。
