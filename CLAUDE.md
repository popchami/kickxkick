# solemuseum 開発ルール

## 操作の自動承認・要確認ルール

### 自動で進める（確認不要）
- コード修正・ファイル編集
- 調査・コード読み込み・検索
- git commit

### 確認必須 ＋ Discord通知

以下に該当する場合は、**まず Discord に「承認待ち」通知を送り、その後チャットで確認を求める**。
承認されてから実行する。（Webhook URL はグローバル設定に保存済み）

- **DBマイグレーション**（スキーマ変更・ALTER TABLE等）
- **git push**
- **大きな設計変更**（アーキテクチャ・データ構造に関わる判断）
- **削除系の操作**（ファイル削除・データ削除・DROP TABLE等）
- **仕様の解釈に迷う場面**

#### 手順（必ずこの順で）
1. Discord に通知（「承認待ち」の内容）
2. チャットで確認を求める
3. ユーザーが承認したら実行
