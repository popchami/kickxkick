# Kick×Kick Backup / Trash Specification v1.1

## 1. ゴミ箱

保持期間: 30日

削除するとゴミ箱へ移動する。

一緒に移動するもの:

- スニーカー情報
- 写真
- 箱写真
- ステッカー
- 履歴
- TOP5情報
- 棚配置情報

30日以内なら復元可能。
30日後に完全削除。

ゴミ箱内もFreeの5足制限にカウントする（30日後の完全削除で初めてカウントから外れる）。

## 2. 復元対象

復元対象:

- スニーカー情報
- 写真
- 箱写真
- ステッカー
- 履歴
- TOP5情報
- 棚配置情報

## 3. バックアップ

Premium限定。

形式:

```text
.kkb
```

保存内容:

- スニーカー
- 写真
- 箱写真
- ステッカー
- Collection
- Sticker Board
- TOP5
- 履歴
- 設定

## 4. 保存先

保存先はユーザー選択。

- Google Drive
- OneDrive
- Dropbox
- 端末

## 5. 世代管理

アプリ内で世代管理しない。

ファイル名には取得日（yyyymmdd）を含める。同日に複数回バックアップした場合のみ、末尾に連番を付ける。

例:

```text
KickxKick_Backup_20260704.kkb
KickxKick_Backup_20260704(1).kkb
KickxKick_Backup_20260704(2).kkb
```

## 6. バックアップ方針

Kick×Kickは写真・ステッカー・棚配置を扱うため、バックアップは重要なPremium価値とする。

バックアップ対象は、ユーザーの展示体験を復元できる単位で保存する。

## 7. 注意

- 世代管理機能は持たない
- 保存先の容量管理はユーザー側に委ねる
- 復元時は既存データとの競合処理が必要
- 画像データの容量に注意する

## 8. 最終バックアップ日時の表示

設定画面などアプリ内に、最終バックアップの実行日時を表示する。

表示形式:

```text
最終バックアップ: 2026/07/04
```

一度もバックアップを取っていない場合:

```text
バックアップがありません
```
